Book 1What Is GazeConnect Pro?

1.1 Problem & Users

Amyotrophic Lateral Sclerosis (ALS) progressively strips away a person's ability to speak, move, and eventually breathe — but it leaves cognition fully intact. Patients are literally trapped inside their own bodies. The ability to communicate is not a luxury; it is a lifeline for expressing pain levels to caregivers, calling for emergency suction, telling loved ones "I love you," or simply saying "I'm cold."

GazeConnect Pro is a medical-grade Augmentative and Alternative Communication (AAC) application designed specifically for ALS/MND patients. It uses the Tobii Eye Tracker 5 — a consumer-grade eye tracking device — to let patients control a full communication interface using only their eye movements. No hands, no voice, no switches required.

💡
Why This Matters

Commercial AAC devices cost $8,000–$15,000 and often require insurance approval that takes months. GazeConnect Pro runs on a standard Windows laptop with a $230 Tobii Eye Tracker 5 — making it accessible to patients who cannot wait or cannot afford medical-grade hardware.

Primary users:

User TypeNeedsALS Stage
ALS PatientsEye-gaze typing, phrase selection, emergency alerts, TTS outputEarly → Late
CaregiversConfigure phrases, manage medical items, monitor fatigue, set dwell speedsN/A
Family MembersPersonalize contacts, add custom phrases, set up home screenN/A

1.2 User Journey

Understanding how a patient uses GazeConnect Pro end-to-end reveals why every architectural decision was made. The system must be zero-friction — every extra millisecond of latency or misplaced button could mean the difference between communicating a medical emergency and suffering in silence.

1.3 High-Level Architecture

GazeConnect Pro is a four-process desktop application. Each process is a separate executable, communicating over local network protocols. This separation ensures that a crash in the word prediction engine never freezes the gaze cursor, and a TTS hang never blocks emergency alerts.

2.4 Auth & Session Flow

GazeConnect Pro is a local-only desktop application with no user authentication, cloud accounts, or remote servers. This is a deliberate choice: ALS patients should never be locked out of their communication device by an expired token, network outage, or forgotten password.

💡
No Auth By Design

All data (customization, learned words, settings) is stored locally on the machine. The only optional network call is to the free Datamuse API for word enrichment — and it has a 300ms timeout with graceful fallback. The app works fully offline.

Session state is managed through React contexts and browser storage:

  • sessionStorage — Keyboard text persistence across screen navigation
  • localStorage — Dwell time settings (gazeconnect_dwell_settings)
  • Electron file I/O — Customization data (phrases, people, medical items, layout)
  • Python in-memory — Learned words, sentence history, gaze calibration profile

2.5 Key Algorithm: Word Prediction Pipeline

The word prediction system is the crown jewel of GazeConnect Pro. It combines five complementary approaches to suggest words with 13.5ms mean latency — fast enough that predictions appear before the patient finishes dwelling on the current key.

Book 3Frontend Layer

3.1 Framework Fundamentals

The frontend is a pure React 18 application — no external UI library, no Tailwind, no styled-components. Every component is built from scratch with inline styles and the custom design system in design.ts. This isn't a style preference — it's a medical necessity. Eye-gaze interfaces need pixel-perfect control over button sizes, hit zones, and visual feedback that no generic UI library provides.

⚠️
Why No UI Library?

Generic UI components assume mouse/touch input with hover states, focus rings, and click handlers. GazeConnect needs dwell-based selection with onset delays, progress rings, magnetism, and cooldowns — none of which exist in Material UI, Chakra, or Ant Design.

3.2 Entry Point & Routing

The app uses a single-screen state machine instead of React Router. There are no URLs to navigate — the patient looks at a navigation button, dwells on it, and the screen state changes. This is simpler, faster, and eliminates any browser history issues.

3.6 Core Gaze Components

The heart of the UI is the gaze interaction system. These components implement the two-stage dwell selection process that makes eye-gaze typing possible.

GazeButton — The Foundation

Every interactive element in the app is a GazeButton. It implements a research-backed two-stage dwell process:

  1. Onset delay (150–450ms, configurable per ALS stage) — Hidden timer that prevents jitter-triggered selections. The user sees nothing during this phase.
  2. Visible dwell (900–2000ms, context-dependent) — A progress ring fills around the button. When complete, the action fires.
// src/components/core/GazeButton.tsx — Key props
interface GazeButtonProps {
  dwellTime?: number;        // Override default dwell duration
  dwellCategory?: string;    // 'navigation' | 'keyboard' | 'emergency' | ...
  variant?: string;          // Visual style variant
  size?: string;             // 'xs' | 'sm' | 'md' | 'lg' | 'xl' | 'xxl'
  onDwellProgress?: (progress: number) => void;
  onDwellComplete?: () => void;  // Fires on successful dwell
}

4.3 Word Prediction Engine (5 Layers)

The WordPredictionEngine in python/services/word_prediction.py (164KB) is the largest single service. It implements a five-layer pipeline where each layer contributes candidates that are merged, ranked, filtered, and personalized.

Layer 1: N-Gram Model

Maintains a 3,000+ word vocabulary with frequency rankings. Includes three sub-vocabularies: core (500 common English words), medical (ALS-specific terms like "suction", "tracheostomy", "bipap"), and patient (350+ words learned from the patient's typing history).

Layer 2: Smart Bigrams

Pre-computed conditional probabilities from python/data/smart_bigrams.json (1,339 word pairs, 36KB). Given the previous word, returns ranked next-word candidates. Example: after "want" → {"some": 72, "drink": 40, "eat": 34, "the": 30}.

Layer 3: CIFG-LSTM Neural Fusion

A custom Coupled Input-Forget Gate LSTM model (1.9MB quantized ONNX) with 661-token vocabulary. The "coupled" gate means the forget gate is computed as 1 - input_gate, reducing parameters by ~25% while maintaining quality. Runs via ONNX Runtime with a hard 30ms timeout — if inference takes longer, the pipeline returns n-gram results only.

Layer 4: Datamuse API

Free online API (no key required) that provides semantically related words. Called in the background with a 300ms timeout. Results are cached (500 entries, 1-hour TTL). Entirely optional — the app works fully offline without it.

Layer 5: Patient Personalization

Words the patient has typed before get significant score boosts: 3x for bigram matches, 2x for vocabulary presence. When a word appears in both the n-gram and neural models ("agreement"), it gets an additional 15% bonus. The SessionTopicTracker provides decay-weighted topic boosts based on recent conversation context.

4.6 Safety: Prediction Guardrails

Data Flow

// tobii-helper/TobiiGazeHelper/Program.cs
// 1. Initialize Tobii Interaction Library
var host = new Host();
var stream = host.Streams.CreateGazePointDataStream(
    GazePointDataMode.LightlyFiltered  // Minimal built-in filtering
);

// 2. On each gaze point (133Hz):
//    - Normalize to [0,1] range
//    - Apply DPI-aware scaling
//    - Send JSON over TCP to port 5555
stream.GazePoint((x, y, ts) => {
    var msg = JsonSerializer.Serialize(new {
        type = "gaze",
        x = x / screenWidth,   // Normalized [0,1]
        y = y / screenHeight,  // Normalized [0,1]
        timestamp = ts
    });
    tcpClient.Send(msg);
});
