# v17.16 - Embedded Browser Gaze Safety Notes

**Status:** Current implementation notes, updated after the bottom Watch Mode rail and Pause Gaze controls were intentionally removed from the UI.

**Goal:** Make the embedded browser feel as safe as the native AAC screens: no accidental video clicks, no true-fullscreen lockout, and large gaze-friendly controls for YouTube and Quick Search.

---

## Current Scope

v17.16 now covers:

- In-video dwell suppression while a video is playing.
- Playback-state detection from the injected BrowserView script to React.
- True fullscreen auto-exit for gaze safety.
- YouTube play/pause, skip-ad, show/hide-controls, back, and volume commands.
- A side-dock **Full Screen** control that maximizes the YouTube video *inside* the BrowserView without entering true browser fullscreen.
- Quick Search / page-link sidebar with large paginated gaze targets.

v17.16 no longer includes:

- A bottom Watch Mode rail.
- Pause Gaze / Resume Gaze countdown.
- BrowserView height shrink for a bottom rail.

Those controls were removed because they duplicated the top toolbar and consumed valuable vertical video space. Keep the current model: top toolbar for primary video actions, right side dock for scrolling and in-app video maximize.

## Implemented Behavior

### 1. Playback State

The injected BrowserView script exposes:

```js
window.gcGetPlaybackState()
```

Expected shape:

```js
{
  playing: boolean,
  hasVideo: boolean,
  rect: { l: number, t: number, w: number, h: number } | null,
  fullscreen: boolean,
  src: string
}
```

`electron/main.ts` polls this state and emits `webview:playbackState` only on meaningful transitions. `useGazeBrowser.ts` stores it as `videoPlaybackState`.

### 2. In-Video Dwell Suppression

When `videoPlaybackState.playing === true` and the gaze point is inside the active video rect:

- The injected cursor stays visible as a dot.
- The dwell ring does not start.
- No card posterior, onset, or click commit runs for that frame.
- `window.gcState.dwellState` reports `'suppressed'`.
- `window.gcState.events2` receives throttled `dwellSuppressed` events.

The rest of the page remains interactive. This is important: recommendations, page links, side controls, scrollbars, and skip-ad targets must still be usable.

### 3. True Fullscreen Safety

True browser fullscreen is unsafe because it hides app-level gaze controls. If a page enters true fullscreen, the injected script attempts to exit it and emits:

- `fullscreenEnter`
- `fullscreenAutoExited`
- `fullscreenLockoutRisk` only after repeated failures

The visible **Full Screen** button in `WebBrowsingScreen.tsx` is intentionally **not** true fullscreen. It runs `YT_MAXIMIZE_SCRIPT`, which adds the `gazeconnect-youtube-inapp-maximize` class inside the YouTube page, hides YouTube side clutter, and expands the player within the BrowserView. App-level top and side controls remain visible.

### 4. YouTube Controls

Top toolbar:

- Emergency
- Back / Close
- Pause / Play
- Next
- Skip Ad
- Show Controls / Hide Controls
- Show Nav when nav is hidden

Right side dock in Control mode:

- Up
- Scroll / Scroll On
- Full Screen, shown only on YouTube watch pages
- Down

All native controls must remain `GazeButton` targets with `dwellCategory` rather than hardcoded dwell times.

### 5. Quick Search Link Sidebar

Quick Search and general BrowserView pages can show an app-owned link sidebar. It exists so the patient does not need to hit tiny native web links.

Current expectations:

- Link cards are at least 80px tall.
- Large mode uses larger cards and fewer items per page.
- Prev, Refresh, Next, and Large/Compact toggle are gaze targets.
- Selecting a link navigates the BrowserView and disables page gaze during the transition, matching the existing browser-navigation safety pattern.

## Telemetry Handles

Open DevTools for the BrowserView page context, not the main React window, then inspect:

```js
window.gcState.dwellState
window.gcState.events2.slice(-25)
window.gcGetPlaybackState()
window.gcState.events2.filter(e => e.kind === 'fullscreenLockoutRisk')
```

Expected values during normal YouTube playback:

- Looking inside the playing video: `dwellState === 'suppressed'`.
- Looking at recommendations or app controls: normal dwell states.
- True fullscreen attempts: `fullscreenEnter` followed by `fullscreenAutoExited`.
- `fullscreenLockoutRisk` remains empty.

## Manual Verification

Run `.\start-dev.bat --simulate` for mouse-as-gaze checks, then repeat on Tobii hardware.

1. Open a YouTube video and look inside the video frame.
   - Pass: no dwell ring, no play/pause toggle, `dwellState` becomes `'suppressed'`.

2. Look at a recommendation thumbnail or a page link outside the video.
   - Pass: normal dwell completes and clicks the intended target.

3. During an ad, dwell `Skip Ad`.
   - Pass: skip-ad still works and suppression does not swallow it.

4. Use top toolbar actions.
   - Pass: Back, Pause / Play, Next, Skip Ad, Show/Hide Controls all dwell-click reliably.

5. Use side dock Full Screen on a YouTube watch page.
   - Pass: video enlarges inside the BrowserView; app top toolbar and side dock remain visible; true fullscreen is not entered.

6. Force true fullscreen from inside YouTube if possible.
   - Pass: fullscreen exits automatically and telemetry logs `fullscreenAutoExited`.

7. Open Quick Search and show Links.
   - Pass: large link cards, Prev/Refresh/Next, and Large/Compact controls are all easy gaze targets; no bottom rail is present.

8. Regression smoke test.
   - Pass: Home, Keyboard typing "hello", Basic Needs, Phrases, and Compass Map feel unchanged.

## Guardrails For Future Browser Work

- Do not add a bottom control rail unless the BrowserView bounds and vertical space are explicitly redesigned and re-tested.
- Do not add a true fullscreen entry point.
- Do not suppress dwell outside the active video rect; the patient must be able to use surrounding controls.
- Do not tune v17.15 Bayesian constants without telemetry showing the failure.
- Do not move browser dwell into the main app `GazeCursor`; BrowserView content is controlled by the injected cursor.
- Keep app-owned controls outside BrowserView as `GazeButton` components with centralized dwell categories.
